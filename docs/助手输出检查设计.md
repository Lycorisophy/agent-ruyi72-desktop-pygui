# 助手输出检查（output_review）

## 设计冻结（实现依据）

- **原文真源**：会话 `messages.json` 中助手 `content` 字符串；增强结果写入会话目录 **`output_annotations.json`**（不按消息拆文件）。
- **偏移**：`start` / `end` 均为 **UTF-16 码元**，与浏览器 `String`、后端 `src/service/utf16_text.py` 一致。
- **MVP 展示**：开启 `output_review.enabled` 后，助手气泡使用 **Markdown 安全渲染**（`marked` + `DOMPurify`）；引用与存疑以 **气泡下方条** 展示；文内高亮留作后续增强。

## 配置

见 `RuyiConfig.output_review` 与 `config/ruyi72.example.yaml` 注释。

## API（pywebview `Api`）

- `get_message_annotations(session_id?, message_index)`
- `request_output_review(session_id?, message_index, user_context?)`（需 `async_doubt_after_reply`）
- `review_message_section(session_id?, message_index, section_id)`

## 检索工具引用（P3）

`src/service/output_review_sync.extend_citations_from_tools` 将正文 URL 与工具行合并；`build_sync_record` 在存在 `tool_citations` 时调用之。

### 落盘：`messages.json` 中助手消息的 `tool_citations`

- **写入方**：ReAct 成功或中断落盘时，由 [src/agent/react_lc.py](../src/agent/react_lc.py) 的 `collect_citation_rows_from_agent_messages` 解析 LangGraph 原始消息，挂到最后一条 `role: assistant` 上。
- **不进入 LLM**：`messages_for_llm` / `raw_flat_from_stored_messages` 仅保留 `role`+`content`，故不会影响后续轮次上下文。

### 工具输出 JSON 契约（供工具实现方）

工具返回字符串中需含 **JSON 对象**，顶层键 **`citations`**，值为数组；元素为对象，字段：

| 字段 | 说明 |
|------|------|
| `url` | 必填，http(s) 链接 |
| `title` | 可选，展示标题 |
| `start` / `end` | 可选，UTF-16 码元偏移，指向助手正文中的 span；缺省时条目仍出现在侧栏，文内不强行对齐 |

支持两种承载方式（解析顺序：先整段 `json.loads`，若无有效 `citations` 再扫 fenced `` ```json `` 代码块）。

无 offset 时仅侧栏列表展示；与正文 URL 去重按 `url` 由 `extend_citations_from_tools` 处理。

**URL 兜底（无 JSON 时）**：若某 `ToolMessage` 在整段 / fenced 解析后仍无有效 `citations`，且工具名为 `search_history`、`search_memory`、`search_memory_semantic`、`browse_memory` 之一，则对工具正文调用 `extract_urls_as_tool_citation_rows`（规则与正文 URL 提取一致，**不产生 UTF-16 偏移**，仅侧栏）。其它工具（如 `read_file`）不扫 URL，避免误报。

## 合规

存疑与按节检查均为 **辅助提示**，非专业结论；外链使用 `rel="noopener noreferrer"`。
