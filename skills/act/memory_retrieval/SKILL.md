# 记忆检索技能

## 元信息
- name: memory_retrieval
- version: 1.0.0
- description: 通过RAG检索相关记忆并注入上下文
- trigger: 用户询问过去事件时 / 用户说"我记得..."时

## 功能说明
当用户询问关于过去的问题时，通过向量检索找到最相关的记忆片段，并将结果注入AI的上下文。

## 检索示例
- "我昨天吃了什么？" → 检索昨天相关记忆
- "之前我们聊过什么关于..." → 检索相关话题记忆
- "我记得上次你说..." → 检索相关内容

## 输入参数
- query: 用户当前问题
- time_range: 时间范围（可选，默认30天）

## 处理流程
1. 将用户问题转换为向量
2. 在向量数据库中检索 Top-K 相关记忆
3. 结合时间上下文过滤
4. 格式化结果并输出

## 输出格式
```markdown
## 检索到的相关记忆

### [高置信度] {记忆标题}
- 内容: {记忆详情}
- 时间: {发生时点}
- 标签: {相关标签}

### [中置信度] {记忆标题}
...

## AI回复建议
根据以上记忆，自然地回答用户问题。
```

## 置信度等级
- 高置信度 (>0.8): 内容完全匹配用户问题
- 中置信度 (0.5-0.8): 内容部分相关
- 低置信度 (<0.5): 需要AI判断是否相关

## 调用方式
```
调用 memory_retrieval 技能
输入: 用户当前问题
输出: 相关记忆列表和上下文注入建议
```

## 注意事项
- 优先检索高置信度记忆
- 如果没有找到相关记忆，坦诚告知用户
- 不要凭空编造记忆内容
