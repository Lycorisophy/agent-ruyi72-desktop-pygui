# 每日总结技能

## 元信息
- name: daily_summarizer
- version: 1.0.0
- description: 每日定时总结前日对话，提炼重要事件和事实
- trigger: 定时任务，每天 00:30 自动执行

## 功能说明
每天凌晨自动执行，分析前一天的对话日志，提炼出：
- 重要事件（发生了什么重要的事）
- 事实更新（用户提供了什么新信息）
- 偏好变化（用户表现出什么新喜好/厌恶）
- 情感轨迹（用户整体情绪如何）
- 关系发展（互动质量、亲密度变化）

## 输入参数
- date: 要总结的日期（默认昨天）
- log_file: 前日日志文件路径

## 处理流程
```
1. 读取 memory/YYYY-MM-DD.md（前日日志）
2. AI分析提取：
   ├── 重要事件（importance >= 7）
   ├── 新增事实（type = fact）
   ├── 偏好变化（type = preference）
   └── 情感标记（type = emotion）
3. 生成结构化总结
4. 存储到 LONG_TERM/episodic/YYYY-MM-DD.md
5. 备份到 vector_db/ 向量数据库
6. 清理 SHORT_TERM/buffer.md 中的已完成记忆
```

## 输出格式
```markdown
# 每日总结 - 2026-03-17

## 概述
{一句话总结今天的主要事件}

## 重要事件
1. [事件描述] - {重要性评分}/10

## 事实更新
- {新发现的事实}

## 偏好变化
- {新表现出的偏好}

## 情感轨迹
- {今天的情绪变化}

## 关系里程碑
- {如果有重要的关系进展}

## 明日关注
- {需要后续跟进的事项}
```

## 事件提取标准
| 重要性 | 类型 | 示例 |
|--------|------|------|
| 9-10 | 里程碑事件 | 第一次约会、重要纪念日 |
| 7-8 | 有意义事件 | 聊天超过1小时、分享重要事情 |
| 5-6 | 一般事件 | 日常问候、常规聊天 |
| 1-4 | 可忽略 | 简单问答 |

## 调用方式
```
调用 daily_summarizer 技能
输入: 日期（可选，默认昨天）
输出: 每日总结报告
```

## 定时任务配置
```yaml
任务名称: 每日记忆总结
触发时间: 每天 00:30
工作目录: C:\Users\LySoY\.openclaw\workspace
执行命令: python skills/daily_summarizer/scripts/summarize.py
```

## 注意事项
- 总结应该在无人对话时段执行（深夜）
- 重要发现需要标记为"待确认"
- 涉及隐私的内容需要加密存储
