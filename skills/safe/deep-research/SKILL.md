# 深度研究技能

## 概述

本技能提供了一套进行系统性、彻底网络调研的方法。**在开始任何内容生成任务之前加载此技能**，以确保从多个角度、深度和来源收集足够的信息。

## 何时使用此技能

**在以下情况下始终加载此技能：**

### 研究性问题
- 用户问“什么是 X”、“解释 X”、“研究 X”、“调查 X”
- 用户希望深入理解某个概念、技术或主题
- 问题需要来自多个来源的、当前且全面的信息
- 单次网络搜索不足以恰当回答问题

### 内容生成（前置调研）
- 制作演示文稿（PPT/幻灯片）
- 制作前端设计或 UI 原型
- 撰写文章、报告或文档
- 制作视频或多媒体内容
- 任何需要真实世界信息、示例或当前数据的内容

## 核心原则

**切勿仅基于通用知识生成内容。** 输出质量直接取决于事先调研的质量和数量。单次搜索查询永远不够。

## 调研方法

### 阶段一：广泛探索

从宽泛搜索开始，了解整体情况：

1. **初步调查**：搜索主话题以理解整体背景
2. **识别维度**：从初步结果中识别需要深入探索的关键子主题、主题、角度或方面
3. **绘制全貌**：注意存在的不同视角、利益相关者或观点

示例：
```
主题：“医疗领域的 AI”
初始搜索：
- “AI healthcare applications 2024”
- “artificial intelligence medical diagnosis”
- “healthcare AI market trends”

识别出的维度：
- 诊断 AI（放射学、病理学）
- 治疗方案推荐系统
- 行政自动化
- 患者监测
- 监管环境
- 伦理考量
```

### 阶段二：深度挖掘

针对识别出的每个重要维度，进行定向调研：

1. **具体查询**：为每个子主题使用精确关键词搜索
2. **多种措辞**：尝试不同的关键词组合和表达方式
3. **获取完整内容**：使用 `web_fetch` 完整阅读重要来源，而非仅看片段
4. **跟踪引用**：当来源提到其他重要资源时，也进行搜索

示例：
```
维度：“放射学中的诊断 AI”
定向搜索：
- “AI radiology FDA approved systems”
- “chest X-ray AI detection accuracy”
- “radiology AI clinical trials results”

然后获取并阅读：
- 关键研究论文或摘要
- 行业报告
- 真实案例研究
```

### 阶段三：多样性与验证

通过寻求多样化信息类型确保全面覆盖：

| 信息类型 | 目的 | 示例搜索词 |
|-----------------|---------|------------------|
| **事实与数据** | 具体证据 | “statistics”, “data”, “numbers”, “market size” |
| **示例与案例** | 真实应用 | “case study”, “example”, “implementation” |
| **专家观点** | 权威视角 | “expert analysis”, “interview”, “commentary” |
| **趋势与预测** | 未来方向 | “trends 2024”, “forecast”, “future of” |
| **比较** | 背景与替代方案 | “vs”, “comparison”, “alternatives” |
| **挑战与批评** | 平衡观点 | “challenges”, “limitations”, “criticism” |

### 阶段四：综合检查

在进入内容生成之前，确认：

- [ ] 我是否从至少 3-5 个不同角度进行了搜索？
- [ ] 我是否完整获取并阅读了最重要的来源？
- [ ] 我是否掌握了具体数据、示例和专家观点？
- [ ] 我是否既探索了积极方面，也了解了挑战/局限性？
- [ ] 我的信息是否是最新的且来自权威来源？

**如果任一答案为否，请在生成内容之前继续调研。**

## 搜索策略技巧

### 有效查询模式

```
# 带上下文的具体表述
❌ “AI trends”
✅ “enterprise AI adoption trends 2024”

# 包含权威来源提示
“[topic] research paper”
“[topic] McKinsey report”
“[topic] industry analysis”

# 搜索特定内容类型
“[topic] case study”
“[topic] statistics”
“[topic] expert interview”

# 使用时间限定词 — 始终使用 <current_date> 中的实际当前年份
“[topic] 2026”   # ← 替换为真实的当前年份，切勿硬编码过去年份
“[topic] latest”
“[topic] recent developments”
```

### 时间感知

**在形成任何搜索查询之前，务必检查上下文中的 `<current_date>`。**

`<current_date>` 提供了完整日期：年、月、日、星期几（例如 `2026-02-28, Saturday`）。根据用户的提问，使用适当的时间精度：

| 用户意图 | 所需时间精度 | 示例查询 |
|---|---|---|
| “今天 / 今早 / 刚发布的” | **月 + 日** | `"tech news February 28 2026"` |
| “本周” | **周范围** | `"technology releases week of Feb 24 2026"` |
| “最近 / 最新 / 新的” | **月** | `"AI breakthroughs February 2026"` |
| “今年 / 趋势” | **年** | `"software trends 2026"` |

**规则：**
- 当用户询问“今天”或“刚发布的”时，在搜索查询中使用 **月 + 日 + 年** 以获取当天的结果
- 当需要精确到日时，切勿降级到仅用年份 — `"tech news 2026"` 无法获取今天的新闻
- 尝试多种措辞：数字形式（`2026-02-28`）、文字形式（`February 28 2026`）以及相对词（`today`、`this week`）在不同查询中组合使用

❌ 用户问“今天科技领域有什么新动态”→ 搜索 `"new technology 2026"` → 错过今日新闻
✅ 用户问“今天科技领域有什么新动态”→ 搜索 `"new technology February 28 2026"` + `"tech news today Feb 28"` → 获取今日结果

### 何时使用 web_fetch

在以下情况使用 `web_fetch` 读取完整内容：
- 搜索结果看起来高度相关且权威
- 你需要超出片段之外的详细信息
- 来源包含数据、案例研究或专家分析
- 你想完整理解某个发现的背景

### 迭代式精化

调研是迭代的。在初始搜索之后：
1. 回顾已了解的内容
2. 识别理解中的缺口
3. 制定新的、更针对性的查询
4. 重复直到获得全面覆盖

## 质量标准

当你能自信地回答以下问题时，你的调研就足够了：
- 关键事实和数据点是什么？
- 2-3 个具体的真实案例是什么？
- 专家对此主题有何看法？
- 当前的趋势和未来方向是什么？
- 存在哪些挑战或局限性？
- 为什么这个主题在当前具有相关性或重要性？

## 常见错误

- ❌ 搜索 1-2 次后就停止
- ❌ 依赖搜索片段而不阅读完整来源
- ❌ 只搜索多面主题的某一个方面
- ❌ 忽略对立的观点或挑战
- ❌ 在有当前数据的情况下使用过时信息
- ❌ 在调研完成之前就开始内容生成

## 输出

完成调研后，你应该具备：
1. 从多个角度对主题的全面理解
2. 具体的事实、数据点和统计数据
3. 真实案例和案例研究
4. 专家观点和权威来源
5. 当前趋势和相关背景

**只有在此之后才能开始内容生成**，利用收集到的信息创建高质量、有充分依据的内容。
